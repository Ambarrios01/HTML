# Traducido

### ¿Qué es HTML?

1. 
    1. HTML5 (Hypertext Markup Language 5) es un lenguaje de marcado usado para estructurar y presentar documentos de hipertexto en la World Wide Web. Fue la quinta y última gran versión de HTML y ahora es una recomendación retirada del World Wide Web Consortium (W3C). La especificación actual se conoce como *HTML Living Standard*. Es mantenida por el Web Hypertext Application Technology Working Group (WHATWG), un consorcio de los principales proveedores de navegadores (Apple, Google, Mozilla y Microsoft).
2. 
    1. HTML5 se publicó por primera vez de forma abierta el 22 de enero de 2008, con una actualización importante y el estatus de “Recomendación del W3C” en octubre de 2014. Sus objetivos fueron mejorar el lenguaje con soporte para los últimos contenidos multimedia y otras funciones nuevas; mantener el lenguaje fácilmente legible por humanos y, al mismo tiempo, entendido de forma consistente por computadoras y dispositivos como navegadores web, analizadores (*parsers*), etc., sin la rigidez de XHTML; y seguir siendo compatible hacia atrás con software más antiguo. HTML5 está destinado a sustituir no solo a HTML 4, sino también a XHTML 1 e incluso al propio DOM Level 2 HTML.
3. 
    1. HTML5 incluye modelos de procesamiento detallados para fomentar implementaciones más interoperables; amplía, mejora y racionaliza el marcado disponible para documentos, e introduce marcado e interfaces de programación de aplicaciones (APIs) para aplicaciones web complejas. Por las mismas razones, HTML5 también es un candidato para aplicaciones móviles multiplataforma, porque incluye funciones diseñadas pensando en dispositivos de baja potencia.
4. 
    1. Se incluyen muchas características sintácticas nuevas. Para incluir y manejar de forma nativa contenido multimedia y gráfico, se añadieron los nuevos elementos `<video>`, `<audio>` y `<canvas>`; las secciones expandibles se implementan de forma nativa mediante `<summary>...</summary>` y `<details>...</details>` en lugar de depender de CSS o JavaScript; y también se añadió soporte para contenido de gráficos vectoriales escalables (SVG) y MathML para fórmulas matemáticas. Para enriquecer el contenido semántico de los documentos, se agregaron nuevos elementos de estructura de página como `<main>`, `<section>`, `<article>`, `<header>`, `<footer>`, `<aside>`, `<nav>` y `<figure>`. Se introdujeron nuevos atributos, algunos elementos y atributos se eliminaron, y otros como `<a>`, `<cite>` y `<menu>` se cambiaron, redefinieron o estandarizaron. Las APIs y el Modelo de Objetos del Documento (DOM) ahora son partes fundamentales de la especificación HTML5, y HTML5 también define mejor el procesamiento de cualquier documento inválido.

### Historia

1. 
    1. El Web Hypertext Application Technology Working Group (WHATWG) comenzó a trabajar en el nuevo estándar en 2004. En ese momento, HTML 4.01 no se había actualizado desde el año 2000 y el World Wide Web Consortium (W3C) estaba centrando los desarrollos futuros en XHTML 2.0. En 2009, el W3C permitió que expirara la carta constitutiva (*charter*) del Grupo de Trabajo de XHTML 2.0 y decidió no renovarla.
2. 
    1. La Mozilla Foundation y Opera Software presentaron un documento de posición en un taller del World Wide Web Consortium en junio de 2004, centrándose en desarrollar tecnologías compatibles hacia atrás con los navegadores existentes, incluyendo un borrador inicial de la especificación Web Forms 2.0. El taller concluyó con una votación—8 a favor y 14 en contra—de continuar el trabajo en HTML. Inmediatamente después del taller, se formó WHATWG para comenzar a trabajar basándose en ese documento de posición, y también se anunció un segundo borrador, Web Applications 1.0. Más tarde, estas dos especificaciones se fusionaron para formar HTML5. La especificación HTML5 fue adoptada como punto de partida para el trabajo del nuevo grupo de trabajo de HTML del W3C en 2007.
3. 
    1. Ian Hickson (Google) y David Hyatt (Apple), de WHATWG, produjeron el primer borrador público de trabajo de la especificación para el W3C el 22 de enero de 2008.
4. 
    1. Muchos navegadores web lanzados después de 2009 son compatibles con HTML5, incluyendo Google Chrome 3.0, Safari 3.1, Firefox 3.5, Opera 10.5, Internet Explorer 9 y versiones posteriores.
